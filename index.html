7 Principles of Naming things

https://www.namingthings.co/naming-things-principles

Consistency

Each concept should be represented by a single, unique name.

If using terms for grouping items,
ensure the term is identifiable with the object its representing.

Try not to reuse the term across multiple objects to represent different concepts.

Use either singular or plurals.

Understandability

name should describe the concept it represents.

E.g. record_type

Avoid the use of ambiguous names.

Try to describe the concept in relation to the object its attached to.

user_group, log_entry_type, task_category

Catalyst Conventions

Regions

country code + airport code + suffix
e.g. nz-wlg-1
(object-sub-object-suffix)

API endpoints

"api" + country code + airport code + suffix
e.g. api.nz-por-1
object-(object-sub-object-suffix)

Projects

project name + sub project name
e.g. my-company-limited/my-project1/dev (object/sub-object/suffix)
